# Learn Rust

这个仓库用于系统学习 Rust，并把学习过程沉淀成可运行的小项目。

当前仓库已经完成初始化，代码里已经接触到这些内容：

- 结构体
- 函数与返回值
- `String` / `&str`
- 可变借用
- 切片
- `match`
- 随机数与命令行输入

这说明你已经可以从“语法入门”进入“边学边做项目”的阶段了。

## 学习目标

建议按下面的顺序推进：

1. 先掌握 Rust 最核心的思维方式：所有权、借用、模式匹配。
2. 再把常用标准库和错误处理补齐，避免只会写示例代码。
3. 然后开始做命令行项目，建立对模块、工程结构和依赖管理的感觉。
4. 最后再进入并发、异步和 Web 开发。

## 学习路线

### 第 1 阶段：语法基础与控制流

目标：写得出简单程序，并且能看懂大部分入门代码。

重点内容：

- 变量、常量、基本类型
- `if`、`match`、`loop`、`while`、`for`
- 函数、表达式、返回值
- 数组、元组、结构体、枚举
- `Option` 和 `Result` 的基本概念

建议练习：

- 温度转换器
- 斐波那契数列
- 简单计算器

### 第 2 阶段：所有权、借用、切片

目标：真正理解 Rust 和其他语言最大的不同。

重点内容：

- ownership
- borrowing
- mutable / immutable reference
- slice
- `String` 和 `&str` 的区别
- 什么时候移动，什么时候借用

这是 Rust 学习最关键的一关，必须反复练。

`[重点项目]` 猜数字游戏升级版

项目要求：

- 保留你现在已有的猜数字雏形
- 增加多轮游戏
- 增加输入校验
- 增加胜负统计
- 支持重新开始

这个项目重点训练：

- 输入输出
- 循环与分支
- 函数拆分
- `match`
- 基本错误处理

### 第 3 阶段：集合、模块与错误处理

目标：开始写更像“项目”而不是“练习题”的代码。

重点内容：

- `Vec`
- `HashMap`
- 模块拆分
- `use`
- `panic!`
- `Result<T, E>`
- `?` 运算符
- 文件读写

`[重点项目]` 命令行 Todo 程序

项目要求：

- 支持新增任务
- 支持查看任务列表
- 支持完成任务
- 支持删除任务
- 支持把数据保存到本地文件

这个项目重点训练：

- 模块组织
- 结构体设计
- 文件持久化
- `Result` 错误传播
- 命令行参数处理

### 第 4 阶段：泛型、Trait、生命周期、迭代器

目标：开始具备阅读中级 Rust 代码的能力。

重点内容：

- 泛型
- Trait
- `impl Trait`
- 生命周期的基础理解
- 迭代器
- 闭包

这一阶段不要求把生命周期学到很深，但要先建立“它为什么存在”的感觉。

`[重点项目]` mini-grep

项目要求：

- 从文件中搜索关键词
- 支持大小写敏感 / 不敏感
- 支持从命令行传参
- 输出匹配行

这个项目重点训练：

- 生命周期在函数参数中的使用
- `Iterator`
- 字符串处理
- 模块拆分
- 测试编写

### 第 5 阶段：测试、工程化与代码质量

目标：让代码变得可维护，而不是只要能跑就行。

重点内容：

- 单元测试
- 集成测试
- `cargo test`
- `cargo fmt`
- `cargo clippy`
- 文档注释

建议练习：

- 给前面的 Todo 和 mini-grep 补测试
- 给核心函数做边界条件测试
- 尝试重构重复逻辑

### 第 6 阶段：并发与多线程

目标：理解 Rust 在系统编程里的强项。

重点内容：

- `thread`
- `move`
- `channel`
- `Arc`
- `Mutex`

`[重点项目]` 多线程日志统计器

项目要求：

- 读取一个或多个日志文件
- 统计关键词出现次数
- 按线程拆分任务
- 汇总统计结果

这个项目重点训练：

- 线程创建
- 所有权在线程中的转移
- 共享状态与同步
- 性能意识

### 第 7 阶段：异步与 Web 开发

目标：进入当前 Rust 生态最常见的实战方向。

重点内容：

- `async/await`
- `tokio`
- HTTP 基础
- JSON 序列化与反序列化
- 路由、状态、错误返回

`[重点项目]` 简单 REST API 服务

建议技术栈：

- `tokio`
- `axum`
- `serde`

项目要求：

- 提供任务增删改查接口
- 返回 JSON
- 做基本参数校验
- 区分成功和失败响应

这个项目重点训练：

- 异步编程
- Web 框架使用
- 分层组织代码
- 工程化接口设计

## 重点项目优先级

如果你只想抓主线，不想一开始铺得太开，建议按这个顺序做：

1. `[重点项目]` 猜数字游戏升级版
2. `[重点项目]` 命令行 Todo 程序
3. `[重点项目]` mini-grep
4. `[重点项目]` 多线程日志统计器
5. `[重点项目]` 简单 REST API 服务

这 5 个项目基本覆盖了 Rust 初学到进阶最重要的能力。

## 建议学习节奏

可以按 8 到 12 周推进：

- 第 1 到 2 周：语法基础 + 所有权
- 第 3 到 4 周：集合、模块、错误处理 + Todo 项目
- 第 5 到 6 周：Trait、泛型、生命周期 + mini-grep
- 第 7 周：测试、重构、代码质量
- 第 8 到 10 周：并发
- 第 10 到 12 周：异步 + Web

如果时间有限，优先保证“每学一段就做一个小项目”，不要只看知识点。

## 仓库后续建议

后面可以把这个仓库逐步整理成下面这样：

```text
learn-rust/
├── src/
│   ├── main.rs
│   └── bin/
│       ├── stage01_01_variables/
│       │   └── main.rs
│       └── stage01_02_functions/
│           └── main.rs
├── examples/
├── notes/
└── README.md
```

建议约定：

- `src/main.rs`：默认实验入口，适合临时草稿
- `src/bin/`：分阶段练习和小项目入口
- `src/bin/stage01_01_variables/main.rs`：第 1 阶段第 1 个练习
- 命名统一为 `stage{阶段号}_{顺序号}_{主题}/main.rs`
- `examples/`：零散语法实验
- `notes/`：学习笔记和踩坑记录

3. 做完后立刻开始命令行 Todo 项目，不要停留在纯语法练习。
